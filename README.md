# Dz Bot

Небольшой Telegram-бот на Go для получения и демонстрации домашнего задания (WebApp + inline-кнопки).

## Описание

Этот репозиторий содержит простой Telegram-бот, реализованный на Go с использованием библиотеки tgbotapi. Бот показывает меню, открывает WebApp и отвечает на несколько команд/коллбэков, например `/start` и `/new`.

Код — в файле `main.go`.

## Функции

- Отправка приветственного сообщения и inline-кнопок (WebApp + FAQ).
- Обработка команды `/new` с сообщением о релизе и планах.
- Обработка коллбэков из inline-клавиатуры (FAQ, инструкции по добавлению дз и т.д.).

## Требования

- Go 1.18+ (проверьте `go.mod`).
- Токен Telegram-бота.

## Установка и запуск

1. Склонируйте репозиторий или скачайте файлы.
2. Установите зависимости (Go modules):

```sh
go mod download
```

3. Установите переменную окружения с токеном бота (не храните токен в репозитории):

для zsh / bash:

```sh
export TELEGRAM_BOT_TOKEN="<YOUR_TELEGRAM_BOT_TOKEN>"
```

4. В `main.go` по умолчанию токен вставлен в вызов `tgbotapi.NewBotAPI("123123123")` — замените строку на использование переменной окружения или передайте токен напрямую (рекомендуется использовать переменные окружения):

Пример запуска, используя переменную окружения:

```sh
export TELEGRAM_BOT_TOKEN="<YOUR_TELEGRAM_BOT_TOKEN>"
go run main.go
```

Или собрать бинарь и запустить:

```sh
go build -o dzbot
./dzbot
```


## Конфигурация и WebApp

- В коде используется WebApp URL: `https://ryo-del.github.io/dzshka/`. При необходимости поменяйте URL в `main.go` (в объекте `tgbotapi.WebAppInfo`).
- Если вы используете свой WebApp, убедитесь, что он доступен по HTTPS.

## Команды бота

- `/start` — показать приветствие и клавиатуру с кнопками (WebApp, FAQ).
- `/new` — отправляет текст с обновлением версии и планами по развитию.

Кроме того, бот обрабатывает коллбэки из inline-кнопок: `/faq`, `/add_dz`, `/errors`, `/notwork`, `/theme`, `/donate`, `/web`, `/nodz`.

## Разработка

- Рекомендую вынести токен в переменную окружения и добавить простую обработку ошибок при отправке сообщений.
- Для локальной разработки используйте `ngrok` или аналог для тестирования WebApp/callback-ов, если нужно.

## Безопасность

- Никогда не коммитьте реальный токен в репозиторий.
- Ограничьте доступ к логам, если в них выводится конфиденциальная информация.
